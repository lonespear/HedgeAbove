# HedgeAbove — Session Summary (2026-05-09)

## TL;DR

This session merged a Streamlit analytics app (HedgeAbove) and a Telegram cron
alerter (`stock-scanner`, repo `lonespear/tick`) into one codebase, then
progressively built it into a **hedge-fund-grade quant platform powered
exclusively by free data sources**.

**23 commits ahead of `origin/master`. Not pushed yet.**

## Two-pronged deployment (one codebase, two install profiles)

| Profile | Install | Entry | Use |
|---|---|---|---|
| **Lean / Pi cron** | `pip install -r requirements-core.txt` | `python scan.py` (cron) + `python -m hedgeabove.cli` for setup | Headless Telegram alerts. No Streamlit. |
| **Full / Dev** | `pip install -r requirements.txt` (transitively pulls core) | `streamlit run app.py` (+ optionally cron on same box) | Full analytics dashboard + scanner |

**Architectural rule (load-bearing):** the core library
(`hedgeabove/{config,db,indicators,rules,alerts,scanner,cli,backtest,scoring,data}`)
never *hard*-imports Streamlit. The two modules that need caching
(`data/market.py`, `data/universe.py`) use a runtime-aware shim: real
`@st.cache_data` if `streamlit.runtime.exists()`, else a no-op decorator.
So the headless scanner imports cleanly without Streamlit installed.

## Capability matrix

### 1. Watchlist & cron alerter
- SQLite-backed `watchlist_groups`, `alert_rules`, `alerts_fired`,
  `snooze` tables — shared between scan.py and Streamlit
- Telegram delivery with one-fire-per-(symbol, rule, day) dedup contract
- 23 rules:
  - **13 technical** (point-in-time backtestable): rsi_oversold/overbought,
    macd_bullish/bearish_cross, golden_cross/death_cross,
    bollinger_breakout_up/down, volume_spike, high/low_52w_breakout,
    price_above/below
  - **10 fundamental**: pe_below/above, pb_below, dividend_yield_above,
    revenue_growth_above, earnings_growth_above, roe_above, profit_margin_above,
    debt_to_equity_below, analyst_upside_above
- Composite combiners: `all` (AND), `any` (OR), `majority` (>50%)
- Per-ticker snooze (skip alerts for N days)
- Per-rule param overrides via `params_json`
- Watchlist export/import as JSON for dev↔Pi sync

### 2. Signal backtester (`backtest/signals.py`)
- `replay_rule` / `replay_composite` — walk historical bars, evaluate rule(s)
  on every consecutive bar pair, record fires + forward returns at 5/10/20d
- `summarize_*` — per-horizon hit rate, avg, median, std, Sharpe
- `cross_section_summary` — same rule across many symbols, ranked
- **Fundamental rules use SEC EDGAR point-in-time facts** (filing-date
  filtered, amendments deduped to earliest filing) — defeats look-ahead
- CLI: `analyze <SYM> <rule>` with composability:
  - `--rules a,b,c --combine all|any|majority` for composites
  - `--by-regime vix|yield_curve` (FRED-driven regime breakdown)
  - `--by-year` (per-calendar-year + alpha-decay z-score)
  - `--exclude-earnings 5,5` (skip fires near earnings)
  - `--show-fires`, `--param k=v`

### 3. Strategy backtester (`backtest/strategy.py`)
- Daily portfolio simulator: cash + open positions, daily mark-to-market
- `max_concurrent` parameter (default 1 = sequential single-position)
- Position sizing: `equal` (1/max_concurrent) or `inverse_vol`
  (target_vol / realized_vol, clamped 0.5–2.0×)
- Built-in benchmark comparison (default SPY buy-and-hold) — emits
  `alpha_total`, `alpha_ann`, `benchmark_max_drawdown`
- CLI: `strategy <rule>` with all the analyze flags plus
  `--max-concurrent N`, `--sizing equal|inverse_vol`, `--target-vol N`,
  `--exclude-earnings`, `--tearsheet`
- Streamlit Strategy Lab tab with single/composite mode, full tearsheet

### 4. Tearsheet metrics (`backtest/tearsheet.py`)
- `monthly_returns`, `calendar_year_returns`
- `drawdown_series` + `drawdown_summary` (max DD, longest underwater stretch)
- `rolling_sharpe` (252-day window, annualized)
- `beta_to_benchmark` (OLS slope of strategy vs benchmark daily returns)
- CLI: `--tearsheet` flag prints all
- Streamlit: monthly returns heatmap (Y×M grid, red/green gradient),
  drawdown area chart, rolling Sharpe line chart, beta metric card

### 5. Cross-sectional composite scorer (`scoring/composite.py`)
- 9 factors: pe, pb, roe, profit_margin, debt_to_equity, revenue_growth,
  earnings_growth, momentum_12_1 (Asness/Moskowitz), realized_vol_60d
- 6 presets: quality, value, growth, momentum, low_vol, quality_value
- Z-score within universe, weighted-sum composite
- `sector_neutral=True` mode: Z within sector (no over-picking cheap-sector names)
- `--by-sector` mode: top-N per sector
- CLI: `score --preset|--weights k=v --symbols A,B|--universe sp500
  --top N --save-as <watchlist> --sector-neutral|--by-sector`
- Streamlit Strategy Lab "Cross-sectional Scorer" tab

### 6. Walk-forward parameter optimization (`backtest/walkforward.py`)
- For K time-ordered folds: optimize on first half, evaluate on second half
- Aggregates true OOS hit rate / avg return / annualized Sharpe
- Detects overfit (in-sample vs OOS gap) and threshold instability across folds
- CLI: `walk-forward <SYM> <rule> --param-name <k> --param-grid v1,v2,v3`

### 7. Macro / regime layer (`data/macro.py`, `backtest/regime.py`)
- FRED via pandas-datareader: VIX (VIXCLS), 2s10s slope (T10Y2Y),
  unemployment (UNRATE), Sahm rule (SAHMREALTIME) — all free, no API key
- Regime classifiers: vix (low/med/high vol), yield_curve (positive/flat/inverted)
- `analyze --by-regime vix|yield_curve` adds per-regime forward-return tables

### 8. Earnings calendar (`data/earnings.py`)
- yfinance `earnings_dates` (~25 historical quarters per issuer + scheduled)
- `is_within_earnings_window(ticker, date, days_before, days_after)`
- `--exclude-earnings N1,N2` flag on analyze + strategy

### 9. Sector classification (`data/sectors.py`)
- yfinance `info['sector']` and `info['industry']`, lru_cached

### 10. EDGAR point-in-time fundamentals (`data/edgar.py`)
- Wraps `edgartools` (no API key, just SEC fair-use Identity header)
- `get_fundamentals_as_of(ticker, as_of_date, current_price)` returns the
  same dict shape as `data/market.py:get_stock_info` but from XBRL facts
  filtered to `filing_date <= as_of` with amendments deduped to earliest
- Concept selection is data-driven (picks the deepest-quarterly series
  per filer), so e.g. NVDA correctly resolves to `us-gaap:Revenues`
  rather than the alphabetically-earlier `RevenueFromContract...`

## Free data sources used (no paid APIs)

| Source | Used for | Free tier notes |
|---|---|---|
| **yfinance** | Bars (OHLCV), today's fundamentals snapshot, earnings dates, sectors | No key, no rate limit (best effort) |
| **SEC EDGAR XBRL** (via `edgartools`) | Point-in-time historical fundamentals — *the* unlock for valid fundamental backtesting | No key required; SEC asks for an Identity header |
| **FRED** (via `pandas-datareader`) | VIX, yield curve, unemployment, macro series | No key for the public series |

**Deliberately avoided** (research agent flagged as "free trial in disguise"
or too-restrictive): Financial Modeling Prep, Alpha Vantage (25/day on free
now), Polygon.io free tier (5 req/min), NewsAPI (no commercial use).

## Real-data quant findings from this session

These are not toy outputs — every number below came from a live yfinance/EDGAR
pull during smoke-testing.

### Signal hit rates that actually work

- **SPY rsi_oversold (5y, 20d horizon):** 30 fires, **86.7% / 90.0% / 76.7%**
  hit rate at 5/10/20 days, Sharpe ≈ 1. RSI mean-reversion on SPX is
  alive and well.
- **NVDA revenue_growth_above 15% (5y, EDGAR point-in-time):** 767 fires,
  65.3% hit rate at 20d, +6.92% avg fwd return, Sharpe 0.41 — fundamental
  signal actually backtestable now.

### Regime-conditional alpha (the non-obvious finding)

**SPY rsi_oversold over MAX history, split by VIX:**
- **high_vol (VIX > 30):** 53 fires, **84.9% hit rate, +7.20% avg / +6.49% median**
- med_vol (VIX 20–30): 46 fires, 60.9% hit, +0.87% avg
- low_vol (VIX < 20): 10 fires, 80% hit (small sample)

RSI mean-reversion has materially more edge in panic. This is the kind
of regime-conditional signal hidden in aggregate stats.

### Multi-decade per-year breakdown (alpha decay check)

**SPY rsi_oversold per calendar year (since 1993):**
- 2001–2002 (dot-com bust): 83–86% hit, +5.7% / +8.1% avg
- 2009 (post-GFC recovery): **100% hit, +17.75% avg** — perfect timing
- **2020 (COVID crash): 28.6% hit, –5.05% avg** — the failure mode
- 2026 YTD: 100% hit (z=+0.93 vs prior mean 79.4% — strong but not a "surge")

### Strategy backtester (single-position, 5y, 20d hold)

- **rsi_oversold on SPY/QQQ/IWM:** 14 trades, +74.8% total, Sharpe 1.34,
  Max DD –11.9% — beat SPY (–24.5% DD) on risk-adjusted basis though
  underperformed in absolute return.
- **2022 (bear market) returned +10.84% while SPY was –18%** — RSI timing
  saved capital when it mattered most. Beta to SPY: 0.36.

### Multi-position diversification edge

**rsi_oversold on 8-ticker basket, max_concurrent=5:**
- equal-weight: Sharpe 2.09, Max DD –17.7%
- inverse-vol:  **Sharpe 2.13, Max DD –11.0%**

Same returns, ~6.7pt drop in max drawdown — the canonical risk-parity
construction benefit.

### Walk-forward (overfit detection)

**SPY rsi_oversold, threshold ∈ {20,25,30,35,40}, 10y / 5 folds:**
- Best threshold varies 20→40 across folds (instability = overfit risk)
- Aggregate OOS Sharpe (annualized): 0.96
- vs the in-sample-fitted Sharpe of ~1.5 → real overfit gap

## CLI cheatsheet (quick reference)

```bash
# Watchlist & alerter
python -m hedgeabove.cli init                           # seed defaults
python -m hedgeabove.cli watchlist {list,add,add-ticker,remove-ticker,export,import}
python -m hedgeabove.cli rule {list,add,disable,enable,delete}
python -m hedgeabove.cli rule add <group> <type> [--param k=v ...]
python -m hedgeabove.cli scan-once [--group X] [--ticker Y]
python -m hedgeabove.cli rules-available                 # all 23 with docstrings
python -m hedgeabove.cli snooze {add,remove,list}
python scan.py                                           # cron entry

# Signal backtesting
python -m hedgeabove.cli analyze <SYM> <rule> [--period 5y] [--show-fires]
  [--rules a,b,c --combine all|any|majority]             # composite
  [--by-regime vix|yield_curve]                          # macro regime breakdown
  [--by-year]                                             # alpha decay check
  [--exclude-earnings N1,N2]                              # earnings filter
  [--param k=v]

# Strategy backtesting
python -m hedgeabove.cli strategy <rule> --symbols A,B|--watchlist <name>
  [--period 5y] [--hold-days 20]
  [--max-concurrent N] [--sizing equal|inverse_vol] [--target-vol 0.20]
  [--benchmark SPY|--no-benchmark]
  [--exclude-earnings N1,N2] [--tearsheet] [--show-trades]
  [--rules a,b,c --combine all|any|majority]

# Cross-sectional scoring
python -m hedgeabove.cli score --preset quality|value|growth|momentum|low_vol|quality_value
  [--weights roe=0.4,pe=-0.3]
  --symbols A,B|--universe sp500
  [--top N] [--save-as <watchlist>]
  [--sector-neutral] [--by-sector]
python -m hedgeabove.cli presets                          # list presets + factors

# Walk-forward optimization
python -m hedgeabove.cli walk-forward <SYM> <rule>
  --param-name <k> --param-grid v1,v2,v3
  [--period 10y] [--horizon 20] [--folds 5]
  [--score-metric sharpe|hit_rate|avg_return]
```

- **Robust TTM for ORCL-style XBRL quirks** — known limitation in the EDGAR
  layer. Some filers (e.g., ORCL FY18–22) report annual values inside
  91-day-spanning facts, inflating per-quarter sums. Z-score ranking
  absorbs this; absolute growth percentages can be misleading.
- **`analyst_upside_above` and `dividend_yield_above`** not yet backtestable
  (no point-in-time source for analyst targets / TTM dividends).
- **News sentiment overlay** (GDELT free) — adds qualitative dimension.
- **Streamlit UI for walk-forward, regime breakdown, earnings filter,
  alpha-decay** — currently CLI-only.
- **Long-short / market-neutral strategies** — currently long-only.
- **Transaction costs / slippage** — currently 0/0; real-world backtests
  should incorporate.

## What this project is now

A defensible quant research platform that:
1. Runs as a cheap Telegram alerter on a Pi via cron, *or*
2. Hosts a hedge-fund-style Strategy Lab (signal backtesting,
   regime conditioning, factor scoring, walk-forward optimization,
   multi-position simulation, full tearsheet) in your browser.

All on free data. Same SQLite. Same code. One repo.
